### 🛠️ 第一阶段：搭建“数据大动脉” (前端与 Worker 通信)

这个阶段的目标是实现文件从主线程到 Worker 的“盲传”，并确保内存所有权被正确转移。

- **核心任务：**

  - 初始化 Vite + React 19 项目，配置 Tailwind CSS。
  - 实现 `Dropzone` 组件，仅获取 `ArrayBuffer`，严禁在主线程进行任何解析。
  - 创建 `worker.ts`，使用 `postMessage(buffer, [buffer])` 接收数据。

- **给 AI 助理的指令：**

  > “请帮我搭建一个 Vite + React 19 项目。创建一个名为 `FileWorker` 的 Web Worker。在主线程实现一个文件拖拽区，当用户上传文件时，通过 `File.arrayBuffer()` 获取二进制数据，并使用 Transferable Objects 将其所有权转移给 Worker。请确保转移后在主线程检查 `buffer.byteLength` 是否为 0，以验证转移成功。”

### ⚙️ 第二阶段：构造“机械肌肉” (Rust WASM 引擎)

这个阶段要实现 Rust 作为“前台分诊护士”的功能，只扫描 Header，不展开几何体。

- **核心任务：**

  - 初始化 Rust 库，配置 `wasm-pack` 和 `serde`。
  - 实现 `extract_metadata` 函数，利用 `georust` 生态读取 Shapefile/Zip 的头部信息。
  - 实现元数据截断策略（如字段超过 50 个执行截断）并进行数据采样。

- **给 AI 助理的指令：**

  > “请在项目中初始化一个 Rust WASM 模块。实现 `extract_metadata` 函数，接收 `Uint8Array`。使用 Rust 库解析 Shapefile 的 Header。要求：1. 只读取字段名、要素估计数量和 BBox。2. 实现字段截断逻辑，若字段超过 50 个则只取前 50 个并标记 `truncated: true`。3. 为每个字段提取前 3 行数据作为 Sample。”

------

### 🧠 第三阶段：打通“大语言中枢” (LUI 与 AST 翻译)

实现从“人话”到“JSON 指令集（AST）”的翻译，这是产品“魔法感”的来源。

- **核心任务：**

  - 设计 `Command Palette` 指令输入框，并根据回传的 Metadata 生成动态快捷指令 Tag。
  - 编写 LLM Prompt 模板，确保大模型只根据 Metadata 生成 JSON 指令，不接触真实坐标。
  - 在前端实现 AST 安全校验器，只允许白名单内的 `action`。

- **给 AI 助理的指令：**

  > “请为我设计一个 LLM Prompt。输入参数包含：用户自然语言指令、Rust 提供的 Metadata JSON。输出要求：必须是符合我们定义的 `GeoSurgical AST` 格式的 JSON，包含 `action`、`field`、`operator` 等键值对。同时在 React 中根据 Metadata 自动生成快捷指令 Tag，例如检测到坐标系为 WGS-84 时提示‘纠偏至火星坐标’。”

------

### 💉 第四阶段：手术执行与可视化 (指令引擎与渲染)

最后一步，让 Rust 根据指令在本地执行空间计算，并用 OpenLayers 渲染结果。

- **核心任务：**

  - 在 Rust 中实现 `Dispatcher`，根据 AST 动态分发任务（如面积过滤、坐标转换）。
  - 实现进度心跳（Progress Event），每处理一定量数据向主线程汇报。
  - 使用 OpenLayers 的 WebGL 模式渲染处理后的 GeoJSON 结果。

- **给 AI 助理的指令：**

  > “请在 Rust 侧实现 `execute_surgery` 函数。它需要接收 `ArrayBuffer` 和 JSON 格式的 AST。使用 `serde` 反序列化指令，并编写一个分发器（Dispatcher）按顺序执行空间计算任务。每处理 1000 条数据，请通过 `postMessage` 向主线程发送进度百分比。处理完成后返回新的二进制结果用于 OpenLayers 渲染。”
