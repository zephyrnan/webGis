### 🛠️ 第一阶段：搭建“数据大动脉” (前端与 Worker 通信)

这个阶段的目标是实现文件从主线程到 Worker 的“盲传”，并确保内存所有权被正确转移。

- **核心任务：**

  - 初始化 Vite + React 19 项目，配置 Tailwind CSS。
  - 实现 `Dropzone` 组件，仅获取 `ArrayBuffer`，严禁在主线程进行任何解析。
  - 创建 `worker.ts`，使用 `postMessage(buffer, [buffer])` 接收数据。

- **给 AI 助理的指令：**

  > “请帮我搭建一个 Vite + React 19 项目。创建一个名为 `FileWorker` 的 Web Worker。在主线程实现一个文件拖拽区，当用户上传文件时，通过 `File.arrayBuffer()` 获取二进制数据，并使用 Transferable Objects 将其所有权转移给 Worker。请确保转移后在主线程检查 `buffer.byteLength` 是否为 0，以验证转移成功。”

------

### ⚙️ 第二阶段：构造“机械肌肉” (Rust WASM 引擎)

这个阶段要实现 Rust 作为“前台分诊护士”的功能，只扫描 Header，不展开几何体。

- **核心任务：**

  - 初始化 Rust 库，配置 `wasm-pack` 和 `serde`。
  - 实现 `extract_metadata` 函数，利用 `georust` 生态读取 Shapefile/Zip 的头部信息。
  - 实现元数据截断策略（如字段超过 50 个执行截断）并进行数据采样。

- **给 AI 助理的指令：**

  > “请在项目中初始化一个 Rust WASM 模块。实现 `extract_metadata` 函数，接收 `Uint8Array`。使用 Rust 库解析 Shapefile 的 Header。要求：1. 只读取字段名、要素估计数量和 BBox。2. 实现字段截断逻辑，若字段超过 50 个则只取前 50 个并标记 `truncated: true`。3. 为每个字段提取前 3 行数据作为 Sample。”

### 🧠 第三阶段：打通“大语言中枢” (LUI 与 AST 翻译)

实现从“人话”到“JSON 指令集（AST）”的翻译，这是产品“魔法感”的来源。

- **核心任务：**

  - 设计 `Command Palette` 指令输入框，并根据回传的 Metadata 生成动态快捷指令 Tag。
  - 编写 LLM Prompt 模板，确保大模型只根据 Metadata 生成 JSON 指令，不接触真实坐标。
  - 在前端实现 AST 安全校验器，只允许白名单内的 `action`。

- **给 AI 助理的指令：**

  > “请为我设计一个 LLM Prompt。输入参数包含：用户自然语言指令、Rust 提供的 Metadata JSON。输出要求：必须是符合我们定义的 `GeoSurgical AST` 格式的 JSON，包含 `action`、`field`、`operator` 等键值对。同时在 React 中根据 Metadata 自动生成快捷指令 Tag，例如检测到坐标系为 WGS-84 时提示‘纠偏至火星坐标’。”

------

### 💉 第四阶段：手术执行与可视化 (指令引擎与渲染)

最后一步，让 Rust 根据指令在本地执行空间计算，并用 OpenLayers 渲染结果。

- **核心任务：**

  - 在 Rust 中实现 `Dispatcher`，根据 AST 动态分发任务（如面积过滤、坐标转换）。
  - 实现进度心跳（Progress Event），每处理一定量数据向主线程汇报。
  - 使用 OpenLayers 的 WebGL 模式渲染处理后的 GeoJSON 结果。

- **给 AI 助理的指令：**

  > “请在 Rust 侧实现 `execute_surgery` 函数。它需要接收 `ArrayBuffer` 和 JSON 格式的 AST。使用 `serde` 反序列化指令，并编写一个分发器（Dispatcher）按顺序执行空间计算任务。每处理 1000 条数据，请通过 `postMessage` 向主线程发送进度百分比。处理完成后返回新的二进制结果用于 OpenLayers 渲染。”

### ⚙️ 第五阶段：Rust WASM 引擎建设（真正的手术刀）

这个阶段的目标是彻底撕掉左下角的 `WASM MOCK MODE`，在浏览器底层建立起由 Rust 驱动的二进制特征扫描与空间计算阵地。

#### 任务 5.1：WASM 环境搭建与零拷贝桥梁

- **核心逻辑：** 在项目根目录建立 `src-wasm` 独立工程，配置 `wasm-pack`。

- **给 AI 助理的指令：**

  > “请在项目根目录下初始化一个 Rust 库，配置 `Cargo.toml`。引入 `wasm-bindgen`、`serde`、`serde_json` 以及 `serde-wasm-bindgen`。配置编译目标为 `web`。在 Web Worker 中配置 `wasm-pack` 编译出的 `.wasm` 模块加载逻辑，确保主线程传过来的 `ArrayBuffer` 能以 `Uint8Array` 的形式无缝递给 Rust 内存指针。”

#### 任务 5.2：手撕真正的 `extract_metadata`（前台分诊）

- **核心逻辑：** 让 Rust 接管文件流，只扫描 Header 和元数据，不展开庞大的几何体。

- **给 AI 助理的指令：**

  > “请用 Rust 实现 `extract_metadata(buffer: &[u8]) -> String` 函数。利用 `serde_json` 或 GIS 相关解析库，当用户传入 GeoJSON 或 Shapefile 二进制流时，Rust 只读取其头部信息、字段描述和要素预估数量。实现字段截断与采样策略：字段超过 50 个时执行截断（标记 `truncated: true`），并为前 50 个字段每列抽取前 3 行非空数据作为真实 Sample，最后以 JSON 字符串形式返回给 Worker。”

#### 任务 5.3：构建 AST Dispatcher（动态指令分发器）

- **核心逻辑：** 让 Rust 能够解析前端传来的 AST JSON 字符串，并动态按顺序调用计算函数。

- **给 AI 助理的指令：**

  > “请用 Rust 实现 `execute_surgery(buffer: &[u8], ast_json: &str) -> Vec<u8>` 入口函数。使用 `serde` 定义强类型的 AST 结构体（匹配我们规划的 `version` 和 `operations` 数组）。编写一个分发器（Dispatcher），循环遍历 operations，根据 `action` 字段（如 `transform_crs`、`filter_area`）动态路由到具体的清洗函数。如果遇到未知的 action，返回结构化错误。”

#### 任务 5.4：注入真实空间算力与进度心跳

- **核心逻辑：** 引入 `georust` 生态，真正执行坐标转换和过滤，同时向外发送进度。

- **给 AI 助理的指令：**

  > “在 Rust 中引入 `geo` 和 `proj`（或轻量级本地纠偏算法）。实现 `transform_crs`（WGS-84 到 GCJ-02 的非线性纠偏）和 `filter_area`（解析几何面积并根据阈值剔除要素）。在遍历要素循环中，每处理 1000 条数据，利用 `wasm_bindgen` 调用 Worker 的 `postMessage` 触发一次 progress event（发送当前步骤名和百分比），确保 LUI 界面能实时更新执行心跳。”

### 🧠 第六阶段：大模型中枢（The Brain）真实对接

将目前硬编码的 AST 转换为由真实大模型实时生成的动态代理。

#### 任务 6.1：编写系统级 Prompt 与 Schema 锁死

- **核心逻辑：** 让大模型在绝对不看坐标的前提下，精准将人话翻译成我们定义的 AST。

- **给 AI 助理的指令：**

  > “请帮我编写一个严格的系统级 Prompt。该 Prompt 接收两个输入：1. 用户在 Command Palette 中输入的自然语言；2. 阶段五返回的 Metadata JSON（包含字段采样）。约束 LLM 扮演 GeoSurgical 编译器，必须且只能输出符合我们 AST Schema 的 JSON 数组，严禁包含任何自然语言解释或 markdown 格式。如果用户提到了 Metadata 中被截断或不存在的字段，必须输出特定的 `need_field_retrieval` 挂起指令。”

#### 任务 6.2：接入大模型 API 与流式解析

- **核心逻辑：** 在主线程中配置大模型调用层（可对接云端 DeepSeek/Claude API，或本地零成本的 Ollama `localhost:11434`）。

- **给 AI 助理的指令：**

  > “在 React 主线程中实现 `generateAST` 函数，接入大模型服务（优先支持本地 Ollama 或标准 OpenAI 格式 API）。当用户点击‘生成 AST’时，将当前文件的 Metadata 与输入文本喂给大模型，获取返回的 JSON，进行安全校验（确保 action 全在白名单内）后，渲染到 `GeoSurgical AST` 面板，并解锁‘确认执行’按钮。”

### 🗺️ 第七阶段：WebGL 渲染平滑过渡与真实导出

把真数据正式喂给右侧的 OpenLayers，并提供下载。

#### 任务 7.1：OpenLayers 开启 WebGL 渲染

- **核心逻辑：** 当 Rust 吐出几十兆的真实 GeoJSON 后，确保地图层能够吃下十万级要素而不掉帧。

- **给 AI 助理的指令：**

  > “修改右侧的地图组件。当接收到 Worker 传回的真实清洗后的 GeoJSON 二进制数据时，利用 `URL.createObjectURL` 转换为本地流，喂给 OpenLayers 的 `VectorSource`。为了应对大数据量，请将传统的 VectorLayer 升级为 `WebGLVectorLayer`（或使用 OpenLayers 的 WebGL 渲染器），利用用户的显卡进行硬解码和高亮渲染，并加上前后数据对比的图层切换开关。”

#### 任务 7.2：打包本地下载与生命周期释放

- **核心逻辑：** 提供最终成果下载，并立刻释放内存，防止多文件导致浏览器崩溃。

- **给 AI 助理的指令：**

  > “在右侧地图下方实现真实的‘导出’功能。将处理后的二进制数据包装为 `Blob`，触发浏览器原生下载。下载完成后，必须显式调用 `URL.revokeObjectURL` 释放内存，并向 Worker 发送清理信号，重置 TaskContext，确保整个系统的内存占用回到最初状态，严格遵守‘Buffer 所有权单行道’约束。”
